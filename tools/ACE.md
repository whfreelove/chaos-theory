# ACE — Adaptive Critique Engine

ACE learns from critique runs and maintains skillbooks that guide future critic behavior and team coordination.

## Team Guidance Lifecycle

Team guidance lives in `.ace/team/critique.json` and is generated by `ace_learn.py`'s team learning pipeline after critique runs.

### Viewing guidance

```bash
python plugins/tokamak/scripts/run_critics.py <change_dir> --show-team-guidance
```

### States

`status` is `Literal["active", "invalid"]` — no custom values.

- **Active (applied)**: Guidance was acted on. Keep `status: "active"`, update `justification` and `evidence` with what was done + commit ref, increment `helpful` counter.
- **Active (deferred)**: Guidance is valid but not yet actionable. Leave as-is. ACE may reinforce on future runs.
- **Invalid (wrong premise)**: Guidance was based on incorrect assumptions. Set `status: "invalid"`, update `justification` with the correction.

Applied and deferred are distinguished by prose in `justification` (prefix with "Applied:" or "Deferred:"), not by the `status` field. ACE's `.skills()` returns active skills by default, so applied guidance stays visible for reinforcement.

### Example: marking applied

```json
{
  "justification": "Applied: testability bullet removed from Integration eval-criteria-1 across all 4 schemas (commit 67ee304)",
  "evidence": "...(original)... | Resolution: Integration skillbook updated with ACE metadata tracing to this guidance",
  "helpful": 2
}
```

### Example: marking invalid

```json
{
  "status": "invalid",
  "justification": "Invalid premise: truncation was caused by account budget limit, not per-critic token limits."
}
```

## Making Skillbook Changes as a Dev

Edit JSON directly with ACE-compatible metadata:

- Always include: `justification`, `evidence`, `updated_at` timestamp
- Trace to team guidance when applicable (reference the skill ID, e.g., `critic_coordination-00006`)
- Individual critic skillbooks are isolated — they don't reference other critics
- Commit changes atomically with clear messages

## ACE Constraints

| Constraint | Impact |
|---|---|
| `status` is `Literal["active", "invalid"]` | No custom values like "applied" or "deferred" |
| `to_dict()` drops unknown fields | Custom top-level keys need manual re-injection (see `_inject_section_order`) |
| `from_dict()` silently discards unknown fields | Fields not in the Skill dataclass are lost on round-trip |
| `.skills()` returns active skills only | Invalid skills are filtered out by default |
| ACE does NOT use git history | Skillbook state is the sole source of truth |

## File Layout

```
plugins/tokamak/
  .ace/
    critics/{schema}/{critic-slug}.json   # per-critic skillbooks
    team/critique.json                     # team coordination guidance
    artifacts/{artifact}.json              # per-artifact evaluation guidance
  critics.{schema}.json                    # critic configs (files, model, effort)
```

## Propagation

`ace_learn.py --propagate` copies learned skills across schemas:

- Only copies non-seed skills (id != `eval-criteria-1`)
- Manual changes should be propagated to all relevant schemas
